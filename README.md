# The Attention Trap: How Phones Capture Our Minds
Live Site: https://jvsousa35.github.io/phone-dependency-storytelling/

This project expands on my midterm website about phone dependency by applying scrollytelling and spec-driven development. The homepage tells a story through scrolling, while the linked pages explain the psychology behind phone addiction and ways to take back control.

## Project Overview
This project is a digital museum exhibition about phone dependency. The museum explores how smartphones became central to everyday life, how persuasive design encourages repeated checking behavior, and how dependency affects attention, mood, productivity, and identity.

The goal was not to build a generic informational website. The goal was to create a curated exhibit with a narrative sequence, artifact framing, and a clear visitor journey.

## Museum Concept
**Title:** The Attention Trap: How Phones Capture Our Minds  
**Topic:** Phone dependency and the attention economy  
**Core Question:** How did a useful communication device become a system that shapes our routines, emotions, and attention?

## Intended Audience
- Students
- Young adults
- General internet users
- Visitors interested in psychology, design, media, and digital culture

## Required Design Framework
### 1. Design Style — Swiss
The site uses a Swiss-inspired design system:
- clean grid structure
- strong typography
- limited color palette
- high contrast
- consistent spacing and order

This style supports the museum's educational tone and helps the exhibit feel serious, intentional, and curated.

### 2. Cialdini Principle — Curiosity
The museum uses curiosity to keep users moving through the exhibition. Each page reveals a new layer of the subject:
- first the device
- then the design system
- then the behavioral loop
- then the consequences
- then the solutions

This pacing is intended to make visitors ask, “What happens next?” and continue exploring.

### 3. Brand Archetype — Sage
The brand archetype is **The Sage**. This archetype guided:
- the calm, educational tone
- the reflective language
- the research-driven framing
- the emphasis on understanding rather than blaming

## AI Orchestration Process
This project was developed using a structured process rather than a one-shot prompt.

### Process used
**Spec → QA → Sprint → QA → Implement → QA**

### Step 1 — Spec
Defined:
- museum concept
- educational goal
- audience
- page structure
- design style
- persuasion principle
- archetype
- source plan

### Step 2 — QA
Reviewed the topic against the assignment goals:
- personally meaningful
- globally relevant
- strong enough for a museum framing
- easy to support with credible sources

### Step 3 — Sprint
Drafted content page-by-page:
- homepage introduction
- exhibit sequence
- behavioral interpretation
- solutions gallery
- process documentation

### Step 4 — QA
Checked whether the site was actually functioning like an exhibit:
- Does it have narrative flow?
- Are the artifacts meaningful?
- Does the visual style feel unified?
- Does the persuasion principle show up in the visitor journey?

### Step 5 — Implement
Built the full HTML/CSS site and README documentation.

### Step 6 — QA
Final review of:
- clarity
- coherence
- page sequence
- visual consistency
- museum feel
- assignment alignment

## Smithsonian Curator Review Summary
The site was reviewed as if it were being considered by a curator designing a public-facing digital exhibition.

### Strongest elements
- Timely and socially relevant subject
- Clear narrative arc from invention to dependency to reflection
- Strong educational value
- Good use of a disciplined design style
- Topic is personally relatable for almost every visitor

### Improvements identified and applied
- Strengthened page sequencing so the museum feels curated
- Reframed sections as exhibits instead of generic content blocks
- Improved artifact language and interpretive labels
- Tightened layout and typography for a cleaner exhibition tone
